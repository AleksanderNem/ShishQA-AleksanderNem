DNS, можно сказать, что это механизм преобразования псевдонима сайта. Т.е. перевод понятного для человека имени (сайта), например, google.ru в понятный для компьютера IP-адрес. Человек делает запрос на понятном ему языке, например, ( www.google.ru), а сервер DNS преобразует его в понятный для машины ip адрес, например, (123.132.31.12).

HTTP протокол, позволяющий людям общаться в интернете между собой, получать различные ресурсы, например HTML-документы, просматривать фото и т.д. В основе данного протокола концепция: клиент (чаще браузер)- сервер.
Протоколы - это стандарты, определяющие формы представ. и способы пересылки сообщений, правила совместной работы различного оборудования в сетях.

Разница между HTTP & HTTPS в шифровании данных
HTTP используется для незашифрованного обмена открытым текстом. Переданные данные могут быть перехвачены и прочитаны человеком в открытом виде. ( логины и пароли могут быть прочитанны сторонним человеком).
HTTPS используется как зашифрованный вид связи SSL / TLS (это безопасное расширение HTTP).
SSL (Secure Sockets Layer)-тип цифровой безопас.,которая позволяет зашифровать связь между сайтом и браузером.
TSL (Transport Layer Security)- обеспечивает безопасность так же, как и SSL(SSL устарел).

Различие между HTTP1.1 и HTTP2 является двоичный уровень кадрирования.
HTTP1.1 хранит все запросы и ответы в простом текстовом формате
HTTP2 использует двоичный уровень кадрирования для инкапсуляции (упаковка данных) всех сообщений в двоичном формате, при этом сохраняя HTTP (методы, заголовки).
Преобразование сообщений в двоичные позволяет HTTP2 применять новые подходы к доставке данных, недоступные в HTTP1.1.

HTTP0.9 использовался только метод GET.

Различие между HTTP1.0 и HTTP1.1
HTTP1.0 может поддерживать длинные соединения, установив заголовок соединения: keep_Alive
HTTP1.1 несколько запросов и ответов могут выполняться одновременно, т.е. конвейерная обработка
HTTP1.0 считает, что каждый сервер привязан к уникальному ip, поэтому заголовок запроса не содержит имени хоста (поле HOST)
Виртуальные хосты могут иметь несколько виртуальных хостов на физическом сервере, и они имеют один и тот же IP-адрес.
HTTP1.1 добавляет код состояния 100.

HTTP3 может еще больше улучшить процесс передачи, поскольку потерянные пакеты в этом случае влияют только на непосредственно затронутые транзакции.
HTTP3 использует то же, что и HTTP/1.1 и HTTP/2. Поддерживает те же операции get & post, коды ответа... Но HTTP3 использует другой транспортный протокол... работает только на QUIC.
QUIC - новый сетевой транспортный протокол.
